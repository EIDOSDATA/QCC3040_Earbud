kse.kymera.capability.capability_base module¶

Capability Base class

class kse.kymera.capability.capability_base.CapabilityBase(cap, kymera, *_, **__)[source]¶

Bases: abc.ABC

Capability base class. Should be subclassed by every capability extension.

This builds an interface on top of kymera to handle capabilities and operators. kymera interface is a single interface of multiple operators while CapabilityBase is an instance that handle a single operator.

# PassThrough is subclassing CapabilityBase
from kats.kymera.capability.common.pass_through import PassThroughCapability
op = PassThroughCapability('pass_through', kymera, input_data_type=1, output_data_type=1)
op.create() # create operator in uut
op.config() # send all configuration messages
op.start()
...
op.stop()
op.destroy()
del op
Parameters

  • cap (str) – Name of the capability

class kse.kymera.capability.capability_factory.CapabilityFactory(platform, kymera)[source]¶

Bases: object

Kalimba Capability factory

This class handles remote representation of capabilities and operators inside kalimba. It supports a factory interface, where capabilities are classified in platforms When this class starts it autodiscovers all the capability types available for a set of platforms. Discovered capabilities should subclass CapabilityBase in a subpackage.

capability = CapabilityFactory(['common'], kymera)
capability.enum_interface()
op = capability.get_instance('BASIC_PASS_THROUGH',
                            ['CHANGE_INPUT_DATA_TYPE', [1]],
                            ['CHANGE_OUTPUT_DATA_TYPE', [1]])
op.create() # create operator in uut
op.config() # send all configuration messages
op.start()
...
op.stop()
op.destroy()
capability.put_instance(op)
Parameters

  • platform (list[str]) – Platforms available

  • kymera (kats.kymera.kymera.kymera_base.KymeraBase) – Instance of class Kymera
