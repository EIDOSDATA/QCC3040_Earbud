class kse.kalsim.uut.kalsim.timer.Timer(kalcmd)[source]¶

Bases: object

Module to handle kalcmd/kalsim simulated timers

It requires a kalcmd instance to operate. kalcmd timers generate a callback when expire in kalcmd background thread, this module attaches to the callback and keeps a list a currently running timers, both absolute and relative.

The user api to this module enables to create timers and check if those timers have expired

Parameters

kalcmd (kats.instrument.Kalcmd.Kalcmd) – Kalcmd instrument

add_absolute_timer(period, callback=None)[source]¶

Add a simulated absolute timer

To check for timer expiration use check_timer_expired or provide a callback func

The callback should be in the format

def timer_fired(timer_id):
    pass

Note that the callback is running in the context of kalcmd background thread so it should not block

Parameters

  • period (float) – Period in seconds

add_relative_timer(period, callback=None)[source]¶

Add a simulated relative timer.

To check for timer expiration use check_timer_expired or provide a callback func The callback should be in the format

def timer_fired(timer_id):
    pass

Note that the callback is running in the context of kalcmd background thread so it should not block

Example code

wait = True

def cb(id):
    print('timer expired')
    global wait
    wait = False

id = timer.add_relative_timer(1.0, cb)
while wait:
    import time
    time.sleep(0.1)
print('execution resumes')
